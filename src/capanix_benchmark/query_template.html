Test Methodology & Documentation (测试任务详解)

1. 任务定义 (Task Definitions)

  • OS Baseline (Cross-NFS Discovery): 在所有配置的 NFS 根上执行原生 find，发现 submission 落点后再递归扫描命中的 submission 子树。
  • OS Integrity (Double Sweep): 首次跨 NFS discovery + 元数据快照 -> 强制 {{integrity_interval}}s 物理等待 -> 第二次跨 NFS discovery + 元数据快照校验。当前测量值为无重试的最优性能。
  • tree (Materialized): 通过 /api/fs-meta/v1/tree 访问物化索引查询路径。
  • on-demand-force-find (Fresh): 通过 /api/fs-meta/v1/on-demand-force-find 访问强制新鲜查询路径；success path 显式携带 group，每组最多一个 inflight，请求只在组间并发。
